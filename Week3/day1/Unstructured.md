Delivery Checker application. We built it using the vibecoding methodology. It is a CLI-based tool designed to centralize and track multi-carrier shipments without a database. Core features include multi-carrier integration, automated status sorting, AI-generated daily summaries, and local file status exports. System constraints exclude user accounts, SMS alerts, live maps, and web-based interfaces. Reported bugs included real-time chat rendering failures, a post-finalization check-in loophole, and broken photo upload displays in the shipment timeline. Work involved an iterative process of prompt-based development, manual debugging of logical edge cases, and GitHub repository management. Final documentation confirms the project successfully runs without a backend database.
